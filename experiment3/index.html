I first wanted to do more with the colors of the drawing. The drawing will always be drawing a line to a randomly placed point, what if each time it does, it can change its color? I created a list of colors (blue, red, and purple) that would change, and each time a line is finished drawing, the random function will pick a random color in the list to change the stroke color for the line. Next, I wanted to control the size of the lines while the lines were being drawn. I accomplished this by allowing the user to press the Q and E keys to control how long the bounds of the lines should be, with Q decreasing the size of the bounds and E increasing the bounds.

Innovate

Finally, we can introduce new or original ideas to our coding process by thinking creatively and pushing the boundaries. I innovated the experiment by allowing the user to have more interactivity to the drawing. Building upon the idea of allowing the user to freely change the line size, I gave an option to toggle between the option to either control or randomly change the stroke after a line is drawn. There's a 50% chance for the line to stay the same, 25% chance to increase the lines's length and then 25% chance to decrease the line's length.
I then added a randomized color scheme changer. Before, I had it set so that each time a line was finished drawing, the color of the lines would change its color within 1 color scheme, which contained: red, blue, and purple. Now, I want to implement a new color scheme, which contains yellow, orange, and green; and have it so with a low percentage that it will change its color scheme. I accomplished this by first creating a boolean variable called "colorScheme2" and setting it to false. If the boolean is equal to "False", the program will use "lineColor1" to draw the lines, else it'll use "lineColor2" to draw the lines. I then created a list with an index of 50, and filled 49 of those with 0s and set the last index to 1. Using this index, I put it in the random function to pick a number in the list and set it to a temp variable. If the temp variable is equal to 0, then the "colorScheme2" variable will not change, but if it's equal to 1 then "colorScheme2" will be changed to the not of "colorScheme2", therefore changing it from true to false and vice versa. With a 1/50 chance of changing a color, the color scheme will change so depending what color scheme it currently is, it'll only use the 3 colors of the current color scheme.
