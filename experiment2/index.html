After imitating the code but changing so it's shape and color, I wanted to expand on the color aspect of the square. What if when players clicked on the mouse key, the colors of the square would change to a random color. In fact, what if every single square was a different random color. It's here where I used the color functions that p5js had to create these randomized color squares, and by clicking on the "C" key, it can swap between all squares in one randomized color or all squares will have a randomized color.

The original code allowed users to use the mouse to control the position and the size of the circle. From there, I changed it to be square then circle, and added code that would change the square's color to all a random color or each square is its own random color. Now, in order to innovate this, I wanted two things added. 1, to allow users to be in a mode where they could rotate the squares and 2, the colors will always be random whenever you start up the page, no matter what.

For the rotate, I had it set so when users would click the "r" key, it would go to the rotate state, where all the squares would rotate by pi/x angle, x being 3.0. In this rotate state, users can increase the x value of pi/x by 0.2 to increase the rotation by click the "e" key, while also choosing the option to decrease the x value of pi/x by the same amount by clicking the "q" key while x is greater than or equal to 3. From testing this, users can create unique art by rotating the squares while also changing its position and size with the movement of the mouse. To change back from rotate mode, users can click on the "r" key again

After finishing this, I went to create a true randomized seed, where no matter when you refresh it, the colors will never start the same as before. In the original code of what I imitated, the seed always started at 0, and then it would "randomly" pick a number when the mouse is clicked to get the next seed number. But that's is not truly randomized, since the seed always started at 0 when refreshed, so the first "random" number or color in this case would always be the same, and continue on. So, in order to get a true unique number, I used p5js's time functions that were available to me. I first got the values of the current month, current day, current year, current hour, current minute, and current second. I then turn these values into strings and concat them together, following this format: Month+Day+Year+Hour+Minute+Second. So if it was 01/20/2024 at 20:54:56, the string that would have been concatenated would've been 1202024205456. Finally, after getting the concatenating this string, I parse it as a number and set the randomSeed using that number. Because time can only progress forward(at least at how we know at the moment), this seed can never be used again as it'll never be 01/20/2024 at 20:54:56 again, meaning the seed will always start of differently. So unless the user manually changes their computer date and time, this means theoretically(cause it is still random), the squares will never start off as the same color as the last refreshed page starting color.
